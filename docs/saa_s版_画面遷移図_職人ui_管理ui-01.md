# SaaS版 画面遷移図（職人UI・管理UI）【ハッカソン版】

本ドキュメントは **「社長ランチごちします freee連携版（SaaS版）」** の
**ハッカソン提出用・簡略構成**としての画面遷移図を定義する。

> 前提方針（重要）
> - freee 連携設定は **.env.local に固定**（画面設定なし）
> - 写真ストレージは **Google Drive に固定**（環境変数＋コード内設定、画面設定なし）
> - UX・業務フロー理解を最優先

---

## 1. 全体思想（ハッカソン版）

- 職人UI
  - 完全ログインレス
  - 初回のみ QR ペアリング
  - 日次操作は「撮って送るだけ」

- 管理UI
  - NextAuth によるログイン必須
  - 税務判断・freee連携判断はすべて管理側
  - **月次集計は手動トリガーのみ**（MVPは自動実行しない）
  - 将来的に **手動トリガー + Cron（毎月自動）** に拡張する余地はあるが、ハッカソン版では実装しない

---

## 2. 職人UI 画面遷移（ログインレス）

### 2.1 初回のみ：端末ペアリング

```
[QRコード読み取り]
        ↓
[端末登録確認]
        ↓（登録）
[ペアリング完了]
```

**QRコード読み取り**
- 管理者が発行した token 付きURL

**端末登録確認**
- 「この端末を登録しますか？」
- 登録ボタンのみ
- 登録時に、サーバーが `device_secret` を払い出し **HTTP-only Cookie** として端末に保存する（以後のログインレス識別に利用）

**ペアリング完了**
- 「次回から自動で申請できます」

---

### 2.2 通常利用：弁当申請

```
[申請トップ]
   ↓
[申請入力]
   ↓
[写真撮影/アップロード]
   ↓
[申請確認]
   ↓
[申請完了]
```

**申請トップ**
- 「今日のランチを申請する」ボタン
- ペアリング済み端末は `device_secret`（HTTP-only Cookie）で自動識別される

**申請入力**
- 利用日（デフォルト：今日）
- 現場名（自由入力 or プルダウン）
- 備考（任意）

**写真撮影/アップロード**
- 弁当 or レシート写真

**申請確認**
- 入力内容・写真サムネイル

**申請完了**
- 「申請を受け付けました」
- 注意文言：最終判断は経理が行う

## 3. 管理UI 画面遷移（ログインあり）

### 3.1 ログイン

```
[ログイン]
   ↓
[管理ダッシュボード]
 ├─ 申請一覧
 ├─ ユーザー管理
 ├─ QR発行（端末ペアリング）
 ├─ 月次集計（手動）
 └─ freee連携履歴
```

表示指標：
- 今月の申請件数
- 要確認件数
- 未集計件数

---

### 3.3 申請一覧

```
[申請一覧]
   ↓（選択）
[申請詳細]
```

**申請一覧**
- 利用日
- 現場名
- 申請者（email）
- ステータス（normal / needs_review）

**申請詳細**
- 写真（Google Drive リンク）
- 重複検知結果
- 備考

操作：
- 承認
- 要確認にする

---

### 3.4 ユーザー管理

```
[ユーザー一覧]
   ↓
[ユーザー詳細]
```

**ユーザー一覧**
- フィルタ：pending / active / needs_review / inactive

**ユーザー詳細**
- email
- 最終申請日
- 紐づく端末（ペアリング済み/未）

操作：
- 有効化 / 無効化
- 名寄せ（手動）
- （任意）このユーザー用QRを発行

---

### 3.5 QR発行（端末ペアリング：アプリ専用QR）

> 目的：ログイン操作が難しい職人向けに、初回のみ「端末＝ユーザー」を紐づける。

```
[QR発行一覧]
   ↓（発行）
[QR表示（共有）]
   ↓（職人が読み取り）
[ペアリング完了ログ]
```

**QR発行一覧**
- 対象ユーザー選択（email/名前）
- 状態（未発行/未使用/使用済/期限切れ）

**発行操作（最小）**
- 「ペアリングQRを発行」ボタン

**QR表示**
- QRコード（token付きURL）
- tokenの有効期限表示
- 操作：コピー / 再発行（失効して新token発行）

**QRの中身（例）**
```
https://your-app.com/pairing?token=XXXXXXXX
```

※ ハッカソン版は「画面に表示してスマホで読み取る」だけでOK（PDF出力は省略可）

---

### 3.6 月次集計（手動トリガー）

```
[月次集計]
   ↓
[集計プレビュー]
   ↓
[freee下書き作成]
```

**月次集計画面**
- 対象月選択
- 件数
- 合計金額
- 要確認件数

**集計プレビュー**
- freee に送信される内容の確認

**freee下書き作成**
- ボタン操作のみ
- freee API へ下書き登録

※ 将来的に Cron は補助（未実行通知用途→自動実行）として拡張可能だが、MVPでは実装しない

---

### 3.6 freee連携履歴

```
[freee連携履歴]
   ↓
[連携詳細]
```

- 実行月
- ステータス（成功 / 失敗）
- freee 下書きURL

---

## 4. 省略した画面（ハッカソン判断）

以下は **.env.local 固定**のため画面を持たない。

- freee 連携設定画面
- Google Drive 設定画面
- テナント設定

---

## 5. 設計上の評価ポイント（審査向け）

- 現場（職人）は迷わない UX
- 税務・会計判断は管理側に集約
- 自動化しすぎず「人の判断」を残す
- freee は下書き連携のみ
- 証跡（写真）は必ず辿れる
