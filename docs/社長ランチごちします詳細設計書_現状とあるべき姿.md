## 1. 概要

### 1.1 サービスの目的

現場で働く従業員の食事補助（福利厚生費）を、**写真を撮って送るだけ**で申請・管理し、freee会計に自動連携するSaaS。

### 1.2 本書の位置づけ

| 区分 | 内容 |
|------|------|
| **As-Is（現状）** | 2026年1月時点の実装済みソースコードに基づく |
| **To-Be（あるべき姿）** | プロダクト運用品質に到達するための目標仕様 |
| **ギャップ** | 両者の差分を一覧化し、他のAIや開発者が作業に着手できる粒度で記述 |

## 6. 認証・認可

### 6.1 管理側認証（As-Is）

**方式**: NextAuth.js Credentials Provider

**実装ファイル**: `src/lib/auth.ts`

```
認証フロー:
1. /api/auth/signin でメール+パスワード入力
2. 環境変数 ADMIN_EMAIL / ADMIN_PASSWORD と照合
3. JWTセッション発行
4. session に tenantId="dev", role="admin" を付与
```

**認可チェック**: `src/lib/auth-server.ts`
- `requireAdminSession()` — role="admin" 必須
- `requireRoleSession(roles)` — 指定ロール必須（定義あり、未使用）

**制限事項（As-Is）**:
- 管理者は1人固定（環境変数ベース）
- テナントIDは `DEFAULT_TENANT_ID` 固定

### 6.3 職人側認証（As-Is）

**方式**: デバイスシークレット（HTTP-only Cookie）

**実装ファイル**: `src/lib/worker-auth.ts`, `src/lib/pairing/device-secret.ts`

```
ペアリングフロー:
1. 管理者が /admin/pairing でQRコード/URL発行
2. 職人が /pairing?token=UUID にアクセス
3. サーバーがトークン検証→デバイスシークレット生成
4. HTTP-only Cookie に device_secret 設定（有効期限1年）
5. 以降のリクエストで Cookie から認証
```

**シークレット管理**:
- 生成: `crypto.randomBytes(32).toString("base64url")` — 256bit
- 保存: SHA-256ハッシュ化してDBに保存
- 検証: Cookie値をハッシュ化してDBのハッシュと照合

**Cookie設定**:
```typescript
{
  name: "device_secret",
  httpOnly: true,
  secure: process.env.NODE_ENV === "production",
  sameSite: "lax",
  maxAge: 365 * 24 * 60 * 60  // 1年
}
```

### 8.3 各Server Action 詳細

#### `submitEntry`（職人申請）

```
入力（FormData）:
  entryDate:   string (YYYY-MM-DD, 必須)
  siteName:    string (1文字以上, 必須)
  totalAmount: number (0以上, 必須)
  note:        string (任意)
  photo:       File (任意)

処理:
  1. requireWorkerContext() でデバイス認証
  2. Zod バリデーション
  3. 写真あり:
     a. SHA-256ハッシュ計算
     b. 重複チェック（findDuplicateByPhotoHash）
     c. Google Driveアップロード（uploadLunchPhotoToDrive）
     d. アップロード失敗時 → 写真なしで続行（警告ログ）
  4. createLunchEntry で保存
     - 写真なし or 重複 → review_status: "needs_review"
     - それ以外 → review_status: "normal"
  5. writeAuditEvent("LUNCH_ENTRY_CREATED")
  6. リダイレクト

出力: リダイレクト（/worker/entry?success=1&review=xxx）
```

#### `previewMonthlyExport`（月次プレビュー）

```
入力（FormData）:
  yearMonth: string (YYYY-MM, 必須)

処理:
  1. requireAdminSession()
  2. Zod バリデーション
  3. listEntriesByMonth で対象月の申請取得
  4. aggregateMonthly で集計
  5. upsertPreviewExport でプレビュー保存

出力: リダイレクト（/admin/monthly?yearMonth=xxx）
```

#### `executeMonthlyExport`（freee送信）

```
入力（FormData）:
  yearMonth: string (YYYY-MM, 必須)

処理:
  1. requireAdminSession()
  2. Zod バリデーション
  3. findExportByBatchId でプレビュー確認
  4. プレビューなし → エラーリダイレクト
  5. createDraftDeal（freee API）
  6. 成功 → markExportExecuted
  7. 失敗 → markExportError
  8. writeAuditEvent("MONTHLY_EXPORT_EXECUTED")

出力: リダイレクト
```

#### `completePairing`（ペアリング完了）

```
入力（FormData）:
  token: string (UUID, 必須)

処理:
  1. Zod バリデーション
  2. findValidPairingToken でトークン検証
  3. findEmployeeById で従業員確認
  4. revokeExistingDevices で既存デバイス無効化
  5. generateDeviceSecret() でシークレット生成
  6. hashDeviceSecret() でハッシュ化
  7. createDevice でデバイス登録
  8. markPairingTokenUsed でトークン使用済み
  9. writeAuditEvent("PAIRING_COMPLETED")
  10. Cookie設定（device_secret, httpOnly, secure, lax, 1年）

出力: リダイレクト（/pairing?success=1）
```
